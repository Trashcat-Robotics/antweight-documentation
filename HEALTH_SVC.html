1.0 HEALTH_SVC-0001

HEALTH_SVC shall provide an interface to mark a device as HEALTHY, SICK, or DEAD.

Parent links: SW-0001

Attribute Value
verified-by unverified
verified-date unverified
verified-comment unverified
type Requirement
rationale If a device is acting offnominally, the software manager in charge of that device may elect to mark it as SICK or DEAD to prevent accidental further usage.

1.1 HEALTH_SVC-0002

HEALTH_SVC shall provide an interface to get the health status (HEALTHY, SICK, or DEAD) of a device.

Parent links: SW-0001

Attribute Value
verified-by unverified
verified-date unverified
verified-comment unverified
type Requirement
rationale A module may need to check if a device is healthy prior to requesting some action.
